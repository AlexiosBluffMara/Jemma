================================================================================
IF EXECUTION FAILS
================================================================================

When Command 3 fails, a JSON report is created:

Location: d:\JemmaRepo\Jemma\state\notebook-smoke\notebook_run_report.json

The report contains:
- python_executable: Which Python interpreter was used
- python_version: Full version string
- phases: Status of each phase
  Example: { "setup": "ok", "deps_check": "failed", "model_load": "pending" }
- first_failure: Details of the first error
  - phase: Which phase failed (e.g., "deps_check")
  - code_cell_index: Which code cell (0-9)
  - notebook_cell_index: Notebook cell position
  - traceback: Full Python traceback

Example failure report structure:
{
  "phases": {
    "setup": "ok",
    "deps_check": "failed",
    "model_load": "pending",
    ...
  },
  "first_failure": {
    "phase": "deps_check",
    "code_cell_index": 1,
    "notebook_cell_index": 3,
    "traceback": "Traceback (most recent call last):\n  File ...\nModuleNotFoundError: No module named 'unsloth'"
  }
}

To read the report:
- Windows: type state\notebook-smoke\notebook_run_report.json
- Or open with any text editor


Common Failures & Recovery:

1. ModuleNotFoundError: No module named 'torch' (or unsloth, datasets, trl)
   → Install missing packages: pip install torch unsloth datasets trl
   → Re-run the notebook command

2. RuntimeError: CUDA is required for this notebook
   → GPU not available or not properly configured
   → Verify: nvidia-smi
   → Update GPU drivers
   → Consider running in WSL2

3. CUDA out of memory
   → Reduce batch size: set JEMMA_BATCH_SIZE=1
   → Use smaller model: set JEMMA_MODEL_NAME=unsloth/gemma-4-E2B-it
   → Close other GPU applications

4. HuggingFace Hub connection error
   → Verify internet connection
   → If behind proxy: huggingface-cli login
   → Check firewall settings
